# Производственная практика
С 05.11.2024 г. по 02.12.2024 г.
Данная практика связана с вашей выпускной квалификационной работой и является подразделом (параграфом, подглавой и т.п.), которые будут включены в текст вашей ВКР.

В рамках практики должна быть выполнена одна из задач диплома, а именно описание стека технологий, используемых при работе над ВКР. Какие технологии, для чего, почему именно они были выбраны.  

_Пример_: Для темы ВКР "Разработка мобильного приложения "Мое хобби"" рассматриваются как отдельно взятые языки разработки Java, Dart, Kotlin, C# (Описываются возможности применения данных языков именно для мобильной разработки, сильные и слабы стороны и т.п.), так и различные фреймворки (с указанием, почему лучше разрабатывать так, а не с чистого листа). Могут, к примеру, быть сравнены различные методологии при разработке мобильных приложений (каскадная модель, скрам (гидкая разработка), модульная и т.д.), описано, для каких проектов какая лучше подходит и т.д. Могут быть рассмотрены разные ОС, под которые можно/нельзя разрабатывать определённой направленности приложения, либо же можно отметить нюансы кроссплатфоорменной разработки.

Структура практики: 
1. Титульный лист
2. Задание на практику

- Изучение и анализ технологий для разработки устройства EasyPass
- Выбор и обоснование технического стека
- Настройка сред разработки

3. Введение 

Описание предметной области - проблема безопасного хранения паролей и их защищенного ввода.  

3. Цели и задачи практики 

Цель: анализ и выбор оптимальных технологий для реализации устройства EasyPass. Задачи:  

- Сравнительный анализ микроконтроллеров
- Выбор среды разработки для МК
- Анализ технологий создания desktop-приложений
- Настройка сред разработки
- Базовое тестирование компонентов

#### 5.  **Теоретическая часть**

**1. Анализ предметной области**

- **Проблема безопасного хранения паролей:**
    
    - Детальное рассмотрение рисков и угроз, связанных с небезопасным хранением паролей. Виды хранения паролей и статистика, кто какими методами паролей пользуется
    - Статистика кибератак, связанных с компрометацией паролей.
    - Психологические аспекты поведения пользователей при управлении паролями.
    
- **Необходимость баланса между безопасностью и удобством:**    
    - Теоретическое обоснование важности доступности паролей при сохранении высокого уровня безопасности. Привести подсчет количества действий, необходимых для вставки паролей с помощью различных хранилищ паролей
    - Исследование последствий неудобства в использовании защитных средств (например, склонность пользователей упрощать пароли или хранить их в незащищенных местах).

**2. Обзор существующих решений**

- **Программные менеджеры паролей:**
    
    - Преимущества и недостатки популярных решений (LastPass, 1Password, Bitwarden).
    - Анализ проблем с удобством использования и безопасностью.
- **Аппаратные решения:**
    
    - Описание существующих аппаратных хранилищ паролей (YubiKey, Mooltipass, OnlyKey).
    - Сравнение их функциональности и удобства использования.
    - Обзор ценовых категорий и доступности для массового пользователя.
- **Выводы из анализа:**
    
    - Идентификация недостатков существующих решений.
    - Подтверждение необходимости разработки нового решения, учитывающего выявленные проблемы.

**3. Теоретические основы решения**

- **Криптографические методы:**
    
    - Описание алгоритмов шифрования, используемых для защиты паролей (например, AES-256).
    - Методы хеширования мастер-пароля (например, bcrypt, SHA-256).
- **Протоколы коммуникации:**
    
    - Теоретическое обоснование выбора протоколов для взаимодействия между устройством и ПК (USB HID, UART).
- **Пользовательский интерфейс и опыт (UX/UI):**
    
    - Принципы разработки интуитивно понятного интерфейса для аппаратных устройств.
    - Влияние удобства использования на безопасность.
- **Методы аутентификации:**
    
    - Обзор различных методов аутентификации (пароли, биометрия, RFID-карты).
    - Обоснование выбора комбинированного подхода (мастер-пароль + RFID-карта).

**4. Обоснование выбора технологий и компонентов**

- **Выбор микроконтроллера STM32F103C8T6:**
    
    - Технические характеристики и их соответствие требованиям проекта.
    - Сравнение с альтернативными микроконтроллерами.
- **Выбор среды разработки:**
    
    - Обоснование использования Arduino IDE для микроконтроллера.
    - Обоснование выбора C# и Visual Studio для разработки настольного приложения.
- **Выбор периферийных устройств:**
    
    - OLED-дисплей, матричная клавиатура, RFID-модуль.
    - Соответствие функциональным требованиям проекта.

---

#### 6.**Практическая часть**

**1. Настройка сред разработки**

- **Установка и настройка Arduino IDE:**
    
    - Инструкции по настройке среды для работы с STM32.
    - Подключение необходимых библиотек и плагинов.
- **Установка и настройка Visual Studio:**
    
    - Создание проекта на C#.
    - Подключение библиотек для работы с USB или COM-портом.

**2. Тестирование взаимодействия компонентов**

- **Подключение и тестирование микроконтроллера:**
    
    - Проверка работоспособности STM32.
    - Загрузка простых скетчей и подтверждение их выполнения.
- **Работа с периферийными устройствами:**
    
    - Тестирование OLED-дисплея (вывод простых сообщений).
    - Тестирование матричной клавиатуры (считывание нажатий клавиш).
    - Тестирование RFID-модуля (считывание меток).
- **Коммуникация между устройством и ПК:**
    
    - Реализация базового обмена данными через USB или COM-порт.
    - Отправка и прием простых сообщений для подтверждения связи.

**3. Базовая проверка функционала**

- **Реализация простых функций на микроконтроллере:**
    
    - Вывод текущего состояния на дисплей.
    - Обработка ввода с клавиатуры для навигации по меню.
- **Создание прототипа настольного приложения:**
    
    - Интерфейс для подключения к устройству.
    - Отображение полученных данных от микроконтроллера.
- **Тестирование шифрования и хранения паролей:**
    
    - Реализация простого шифрования данных на микроконтроллере.
    - Проверка корректности шифрования и расшифровки.

**4. Анализ результатов и корректировка плана**

- **Оценка работоспособности выбранных технологий:**
    
    - Анализ успешности тестирования каждого компонента.
    - Выявление возможных проблем и ограничений.
- **Корректировка технических решений:**
    
    - При необходимости, пересмотр выбора библиотек или методов.
    - Планирование дальнейших шагов для реализации полного функционала.

---

**Обоснование содержания частей отчета:**

- **Теоретическая часть** основывается на анализе проблемы, изучении предметной области и обосновании выбора технических решений. Она включает в себя глубокое понимание существующих решений, выявление их недостатков и теоретическое обоснование того, как ваш проект «EasyPass» решает проблему баланса между безопасностью и удобством.
    
- **Практическая часть** посвящена непосредственной реализации поставленных задач. В ней описывается процесс настройки сред разработки, тестирования компонентов и реализации базового функционала устройства. Практическая часть демонстрирует, как теоретические знания были применены на практике для достижения целей практики.
    

---

**Рекомендации по оформлению:**

- **Используйте литературные источники** для подкрепления теоретических разделов, включая научные статьи, книги и официальную документацию производителей оборудования.
    
- **Включите диаграммы и схемы** в теоретическую часть для иллюстрации архитектуры системы и процессов.
    
- **В практической части** добавьте фотографии прототипа, скриншоты программного обеспечения и фрагменты кода с пояснениями.
    
- **Заключение** должно содержать выводы о проделанной работе, достигнутых результатах и планах на дальнейшую разработку проекта.

7. Заключение 

Результаты анализа и тестирования, выводы по выбранным технологиям  

8. Список использованных источников 

Документация по STM32, Arduino IDE, Visual Studio и др.

# Практика по получению первичных навыков научного исследования
с 03.12.2024 г. по 16.12.2024 г.

ПпППННУ  связан с темой ВКР студента. В этом случае в [отчет по ПпППННУ](https://lms.uni-dubna.ru/mod/assign/view.php?id=89905 "отчет по ПпППННУ") включается теоретическая или теоретико-аналитическая глава из диплома. Это может быть анализ предметной области, предварительная подготовка датасетов перед обучением программы, выявление проблемы заказчика, сравнительный анализ систем/похожих решений/конкурентов и т.п.  
Тема ПпППННУ  в этом случае определяется руководителем ВКР.

При варианте выполнения НИР/ПпППННУ, когда он напрямую связан с темой ВКР, выполняемые в ходе исследования задачи определяет научный руководитель. Однако, обязательно должно быть: описание предметной области тематики ВКР, выделение целей и задач ВКР, подробное описание теоретической составляющей ВКР, всех методов анализа или обработки данных предпроектной части ВКР, может быть включен раздел выявления и опсиания первичных требований к разрабатываемому/внедряемому ПО/модулю/сайту.

# Технологическая практика
с 17.12.2024 г. по 30.12.2024 г.
Данная практика посвящена изучению платформы Kaggle. Студентам предстоит познакомиться с технологическими возможностями Kaggle, опробировать базовые Python-команды и алгоритмы машинного обучения, применяемые при анализе разнородных данных в задачах классификации или регрессии.

https://www.kaggle.com/  

_Kaggle_ — это платформа конкурсов по исследованию и анализу разнородных данных, а также платформа для специалистов по обработке данных и машинному обучению. 

1) Авторизоваться нужно через имеющийся _google_-аккаунт 

2) Далее откроется доступ к различным наборам данных (иначе — датасетам) в разделе _dataset_s. На _Kaggle_ можно найти и реальные задачи и уже подготовленные датасеты для отработки алгоритмов машинного обучения.
  
### Задание на Технологическую (проектно-технологическую) практику:
Студентам необходимо:  
1) зарегистрироваться на платформе https://www.kaggle.com/  
2) выбрать датасет для анализа   
3) загрузить датасет и провести анализ признаков:  
- построить распределение 2-3 признаков  
- построить матрицу корреляции (heatmap) и установить, какие признаки коррелируют между собой сильнее, выбрать 2-3 пары признаков  
- проверить выбранные коррелирующие пары на наличие выбросов  
- сделать выводы, какие пары данных подходят для следующих этапов анализа данных (решения задачи классификации или регрессии)

Для студентов подготовлен [пример выполнения задания](https://lms.uni-dubna.ru/mod/resource/view.php?id=89923 "пример выполнения задания"), для своего отчета по практике копировать пример нельзя!
